ElGamal Alogrithm

The ElGamal Algorithm is the public key encryption similar to RSA but they depend on different key of security. [1]

  • Security of RSA depends on the difficulty of factoring large integers.
  • Security of ElGamal depends on the difficulty of computing discrete logs in a large prime modulus.

It's encouraged to use different random k for different messages. Otherwise, we can see the pattern in the ciphertext. For example, we use the same k for 2 messages m1 and m2 and the resulting ciphertext pairs are (r1, t1) and (r2, t2). Then, t1t2 = m1m2. m1 could be computed if we already known m2.
However, this can be the advantage of ElGmal because we can generate random k for each message and the ciphertext will be different even for the same message. We cannot see the pattern of the message from the ciphertext.
The disadvantage of ElGamal is that the ciphertext is twice as long as the plaintext. [2]
